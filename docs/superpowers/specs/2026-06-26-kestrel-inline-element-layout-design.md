# Kestrel — 인라인 요소 레이아웃: 설계 문서

- 날짜: 2026-06-26
- 상태: 승인됨
- 범위: `layout.rs`의 인라인 텍스트 레이아웃 확장.

## 1. 맥락 / 문제

현재 `layout_text`는 블록 박스의 **직접 텍스트 자식만** 그린다. 그래서:
- 중첩 인라인 요소(`<a>`, `<span>`, `<b>` 등) 안의 텍스트가 누락된다.
- 인라인 요소만 든 블록은 직접 텍스트가 없어 **높이 0** → 다음 블록과 겹쳐 그려진다(CERN 첫 사이트 상단 겹침).

## 2. 목표

중첩 인라인 요소 안의 텍스트도 수집해 렌더하고, 인라인만 든 블록의 높이를 올바르게 계산한다.

## 3. 접근

`LayoutBox::layout`에서 분기:
- **블록 자식이 있으면**(`children` 비어있지 않음) → 블록 컨테이너(현행 `layout_children`).
- **블록 자식이 없으면** → 인라인 컨테이너 → `layout_inline`.

`layout_inline`:
1. `collect_runs(styled_node, color, px)`: 자식을 문서 순서로 재귀 순회. 텍스트 노드는 `(text, color, px)` 런으로. 인라인 요소는 그 요소의 `color`/`font-size`(없으면 상속)로 재귀. 블록/`none` 요소는 스킵.
2. 런들을 문자 단위 `(char, color, px)`로 펼치고 공백 기준으로 **단어**로 토큰화(공백이 단어 경계 → `foo<b>bar</b>`는 한 단어로 보존).
3. 콘텐츠 폭에서 워드랩, 각 글자를 자기 `px`/`color`로 `GlyphInstance` 추가. 줄간격·베이스라인은 박스 기본 `font-size` 메트릭 사용.
4. 박스 높이 = 줄 수 × 줄간격.

## 4. 범위 / 비범위

**범위**: 인라인 요소 텍스트 수집 + `color`/`font-size` 적용(상속 흉내), 워드랩.

**비범위**: 인라인 박스 자체의 배경/테두리/패딩, 인라인 내 블록 혼합(anonymous box), 줄 내 혼합 크기 수직정렬 정밀화, 인라인 `<img>`.

## 5. 테스트 / 검증

- 단위: `<p>a <span>b</span> c</p>`가 글리프 3개 이상 생성(인라인 텍스트 수집). 인라인만 든 블록(`<div><a>link</a></div>`)의 height > 0(겹침 해소). 기존 레이아웃/텍스트 테스트 유지.
- 통합: CERN 첫 사이트를 다시 렌더 → 상단 겹침 해소 + 링크 텍스트 표시(헤드리스).

## 6. 완료 기준

1. 인라인 요소 안 텍스트가 렌더된다.
2. 인라인만 든 블록이 0 높이로 겹치지 않는다.
3. 기존 모든 테스트 통과.
